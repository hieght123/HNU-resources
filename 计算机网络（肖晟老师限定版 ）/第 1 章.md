# 1.1 什么是因特网？
## 1.1.1 具体构成描述
### 重要性
★★★
### 要背的边角料
- **端系统 (End Systems)**: 连接到因特网的各种计算设备。也称为**主机 (Host)**。
- **分组交换机 (Packet Switches)**: 转发分组。主要包括**路由器 (Router)**（用于网络核心）和**链路层交换机 (Link-layer switch)**（用于接入网）。
- **通信链路 (Communication Links)**: 由不同类型的物理媒体组成，如同轴电缆、铜线、光纤和无线电频谱；链路传输速率常用 **bps (比特/秒)** 表示。
- **分组 (Packet)**: 端系统把应用数据分段并加上首部字节形成分组；分组交换机根据分组首部信息进行转发。
- **教科书历史数据 [8th Edition]**: 
    - 联网设备数量：约 **285 亿**。
    - 年度 IP (网际协议) 流量：约 **5 泽字节 (Zettabytes / 10²¹ 字节)**。
- **ISP (因特网服务提供商)**: 每个 ISP 自身就是一个由链路和交换机组成的网络。
- **标准**: **IETF (互联网工程任务组)** 发布的 **RFC (请求评论)** 是事实上的标准。
- **标准的重要性**: 标准保证不同厂商、不同系统实现的协议和设备能够互操作；没有统一标准，端系统、路由器和应用程序之间很难正确通信。
 



### 最可能怎么考
- **填空题**: 链路层交换机通常用于 **接入网**，而路由器通常用于 **网络核心**。
- **填空题**: 链路的传输速率通常以 **bps** 为单位。
- **简答题**: 标准对于网络协议为什么重要？
  - **答案**：协议标准规定了报文格式、交互顺序和动作规则，使不同厂商、不同操作系统、不同设备实现的网络实体能够互相理解并正确通信。因特网中常见协议标准通常以 **RFC** 形式发布。
- **简答题**: 描述因特网硬件构成的三个基本要素。
  - **答案**：1. **端系统**：运行网络应用程序的主机，如 PC、手机、服务器。2. **通信链路**：由光纤、铜线、无线电等物理媒介组成，按传输速率传送比特。3. **分组交换机**：包括路由器和链路层交换机，负责接收分组并转发到下一条链路。

## 1.1.2 服务描述
### 重要性
★★
### 要背的边角料
- **分布式应用**: 运行在端系统上。**注意**：因特网应用不运行在网络核心的交换机上。
- **套接字接口 (Socket Interface)**: 一套发送程序必须遵循的**规则集合**，规定了程序请求因特网交付数据的方式。本质上，因特网为应用程序提供的服务就是一个 **API (应用程序编程接口)**。

### 最可能怎么考
- **填空题**: 进程通过一个称为 **套接字 (Socket)** 的接口向网络发送数据。
- **简答题**: 为什么说路由器和链路层交换机不运行应用层代码？
  - **答案**：路由器和交换机是网络核心设备，其主要功能是向下三层（网络层及以下）进行分组转发，不具备处理或解析应用层业务逻辑（如 HTTP (超文本传输协议)、SMTP (简单邮件传输协议)）的能力；分布式应用程序的代码仅在网络边缘的**端系统**中执行。

## 1.1.3 什么是协议？
### 重要性
★★★★★ (必考名词解释)
### 要背的边角料
- **协议定义**: 定义了在两个或多个通信实体之间交换的报文格式和顺序，以及在报文发送/接收或其他事件所采取的操作。
- **核心三要素**: **格式 (Format)、顺序 (Order)、操作 (Actions)**。

### 最可能怎么考
- **名词解释**: 请给出协议 (Protocol) 的准确定义及核心三要素。
  - **答案**：协议定义了在两个或多个通信实体之间交换的报文的**格式 (Format)** 和 **顺序 (Order)**，以及在报文发送/接收或其他事件所采取的**操作 (Actions)**。三个核心要素：1.**格式**：数据与控制信息的结构。2.**顺序**：交换报文的先后顺序。3.**操作**：收到或发送报文后执行的具体动作。

# 1.2 网络边缘
## 1.2.1 接入网
### 重要性
★★★★★
### 要背的边角料
- **主机 (Host)**: 端系统也称为主机，因为它们容纳并运行应用程序；主机可进一步分为**客户 (Client)** 和**服务器 (Server)**。
- **客户 (Client)**: 通常是桌面 PC、移动 PC、智能手机等用户端设备，运行 Web 浏览器、邮件客户端等客户程序。
- **服务器 (Server)**: 通常是性能更强的机器，用于存储和发布 Web 页面、流视频、邮件等内容或服务；大型服务通常部署在**数据中心 (Data Center)**。
- **数据中心 (Data Center)**: 容纳大量主机，数据中心中的主机通常称为**刀片服务器 (Blade Server)**；内部还包括复杂的计算机网络，用于把数据中心内的主机互联；是许多因特网应用和云服务的基础设施。
- **云计算 (Cloud Computing)**: 公司把数据中心中的计算、存储和网络能力作为服务提供给外部用户或企业；典型例子是 **Amazon Web Services (AWS, 亚马逊 Web 服务)**。
- **DSL (数字用户线)**: 利用现有电话线。**DSLAM (数字用户线接入复用器)** 位于中心局 (CO)。特征是**专用链路**和**不对称 (Asymmetric)**（下行速率远大于上行）。利用 **FDM (频分复用)** 分隔数据和电话信号。
- **电缆接入 (HFC)**: 混合光纤同轴。家庭侧使用电缆调制解调器，**CMTS (电缆调制解调器端接系统)** 位于电缆头端。特性是 **共享广播介质**，也常用 FDM 划分下行和上行信道。
- **FTTH (光纤到户)**: 光纤分布体系结构包括 **AON (有源光网络)** 和 **PON (无源光网络)**；AON 本质上是交换以太网，PON 结构包括 ONT (光网络终端)、splitter (分配器) 和 OLT (光线路终端)。
- **WiFi (无线局域网技术，802.11)**: 共享传输速率高达 **100Mbps**。
- **接入网分类**: 住宅接入常见 DSL、电缆/HFC、FTTH；公司/校园接入常见以太网和 WiFi；广域无线接入常见 4G/5G 蜂窝网络。
- **广域无线速率口径**: 4G 实际下载速率可达约 **60Mbps**；5G 固定式无线也开始用于住宅接入。
- **以太网 LAN (局域网)**: 常用于公司、大学和家庭局域网，典型速率从 100Mbps 到 10Gbps 量级，依具体以太网标准而定。
- **住宅接入速率与共享性口径**: 拨号调制解调器最高约 56kbps；DSL 通常可达数十 Mbps，用户到中心局线路偏专用；HFC/电缆可达数百 Mbps，但同一接入段用户共享；FTTH 具有 Gbps 级潜力，光纤路径偏专用或经 PON 共享汇聚。

### 最可能怎么考
- **填空题**: 端系统也称为 **主机 (Host)**，因为它们运行网络应用程序。
- **填空题**: 大型服务器通常被安置在 **数据中心 (Data Center)**；亚马逊提供的云服务称为 **AWS / Amazon Web Services**。
- **简答题**: 客户和服务器分别是什么？
  - **答案**：客户通常是用户直接使用的主机，如 PC、手机，运行浏览器或邮件客户端等客户程序；服务器通常是更强大的主机，用于存储、处理并向客户提供 Web 页面、视频、邮件等服务，大规模服务器常集中在数据中心。
- **填空题**: 电缆因特网接入在电缆头端的设备称为 **CMTS**，其功能类似于 DSL 网络中的 **DSLAM**。
- **填空题**: DSL、电缆/HFC、FTTH 通常属于 **住宅接入**；以太网和 WiFi 常用于 **公司/校园接入**。
- **简答题 1**: 简述 DSL 与电缆接入在介质共享性上的主要区别。
  - **答案**：DSL采用**专用 (Dedicated)** 物理链路，每个住户到中心局(CO)有独立双绞线；电缆接入(HFC)采用**共享 (Shared)** 广播介质，节点下所有住户共享同轴电缆，互相影响。
- **简答题 2**: HFC 网络的“不对称性”是指什么？
  - **答案**：指网络分配给**下行链路 (中心局到家庭)** 的传输速率和频带宽度远大于 **上行链路 (家庭到中心局)** 的速率，这符合大多数家庭用户下载多于上传的流量特征。
- **简答题 3**: HFC 下行信道中会发生碰撞吗？为什么？
  - **答案**：通常不会。HFC 下行方向是由电缆头端向用户广播发送，只有一个发送端；碰撞主要出现在多个用户共享同一上行信道并同时发送的场景。

## 1.2.2 物理媒介
### 重要性
★★★★
### 要背的边角料
- **导引型媒体 (Guided Media)**: 电波沿着固体媒介前行。如**双绞铜线** (最便宜, 常为 UTP (非屏蔽双绞线))、**同轴电缆**、**光纤** (衰减低、难窃听，OC-n 标准速率=$n \times 51.8$ Mbps)。
- **非导引型媒体 (Unguided Media)**: 电波在空气/真空中传播。如地面无线电、卫星。
- **同步卫星**: 距离地面 **36000km**，传播时延约为 **280ms**。
- **地面无线电信道分类**: 可按覆盖范围分为短距离、局域网、广域蜂窝等；特点是会受路径损耗、遮挡、多径传播和干扰影响。

### 最可能怎么考
- **填空题 1**: 同步卫星带来的信号传播时延大约为 **280** 毫秒。
- **填空题 2**: 在物理媒体的分类中，双绞线和光纤属于 **导引型** 媒体，而无线电频段属于 **非导引型** 媒体。
- **填空题 3**: 以太网可以运行在双绞铜线、同轴电缆、光纤等多种 **导引型媒体** 上。

# 1.3 网络核心
## 1.3.1 分组交换
### 重要性
★★★★★
### 要背的边角料
- **报文 (Message)**: 应用程序要发送的原始数据块；源端系统会把长报文划分成较小的数据块。
- **分组 (Packet)**: 源端系统给数据块加上首部后形成的网络传输单位；分组在通信链路和分组交换机之间转发。
- **分组交换机 (Packet Switch)**: 接收分组并把它转发到下一条通信链路的设备，主要包括**路由器 (Router)** 和**链路层交换机 (Link-layer Switch)**。
- **路径 / 路由 (Path / Route)**: 一个分组从发送端到接收端经过的一系列通信链路和分组交换机。
- **存储转发传输 (Store-and-Forward Transmission)**: 分组交换机必须先接收到整个分组，才能开始向下一条链路转发。
- **输出缓存 / 输出队列 (Output Buffer / Output Queue)**: 当输出链路正忙或到达速率超过输出链路速率时，分组在这里等待发送。
- **排队与分组丢失 (Packet Loss)**: 分组到达速率暂时超过输出链路速率时产生排队；若缓存已满，新到达分组会被丢弃。
- **转发表 (Forwarding Table)**: 分组交换机根据分组首部中的目的地址查表，决定应转发到哪条输出链路。
- **路由选择协议 (Routing Protocol)**: 用于自动设置和更新转发表，使分组能够沿合适路径到达目的地；具体机制后续章节再学。
- **统计多路复用 (Statistical Multiplexing)**: 分组交换按需动态共享链路带宽，而非预先分配；与电路交换对比时最常考。
- **存储转发计算模板**: 若只有 1 台分组交换机、2 条速率均为 $R$ 的链路，忽略排队/传播/处理时延，发送 1 个长度为 $L$ 的分组需要 **$2L/R$**。
- **连续发送多个分组模板**: 若经过 $N$ 条速率相同为 $R$ 的链路，连续发送 $P$ 个长度为 $L$ 的分组，忽略其他时延，总时间约为 **$(N+P-1)L/R$**。
- **报文分段**: 长应用层报文会被切成多个较小分组。优点是可在多条链路上流水线传输，显著降低存储转发总时延；缺点是每个分组都要带首部，增加首部开销，且接收端需要重组，丢失任一分组都可能影响完整报文恢复。
- **统计多路复用概率口径**: 若每个用户独立以概率 $p$ 活跃，则“同时活跃用户数”服从二项分布；考试通常只需说明超过链路容量上限的概率很小。

### 最可能怎么考
- **填空题**: 源端系统把应用层**报文 (Message)** 划分并加上首部形成 **分组 (Packet)**。
- **填空题**: 分组从源到目的地经过的一系列通信链路和分组交换机称为 **路径 / 路由 (Path / Route)**。
- **填空题**: 分组交换机中等待输出链路发送的缓存称为 **输出缓存 / 输出队列**；缓存满时会发生 **分组丢失**。
- **填空题**: 路由器根据 **转发表** 决定分组的输出链路，转发表通常由 **路由选择协议** 设置和更新。
- **填空题**: 分组交换网络中，由于用户的流量是突发的，链路资源是按需共享的，这种资源共享方式被称为 **统计多路复用**。
- **简答题**: 描述路由器中发生排队时延和丢包的根本原因。
  - **答案**：当多个输入链路的分组**同时到达**并需要转发到同一条输出链路时，如果分组的**到达速率暂时超过了输出链路的传输速率**，就会在路由器的输出缓存中产生**排队时延**；当缓存被填满后，后续到达的分组无处存放，就会被路由器**丢弃 (丢包)**。
- **计算题**: 1 台存储转发交换机连接发送主机和接收主机，两段链路速率均为 $R$，分组长度为 $L$，忽略其他时延，端到端时延是多少？
  - **答案**：交换机必须先完整收到分组再转发。第一段链路传输需要 $L/R$，第二段链路传输也需要 $L/R$，总时延为 **$2L/R$**。
- **计算题/应用题**: 分组交换 vs 电路交换 容量对比计算（1Mbps链路，每个用户需100kbps，活跃概率0.1）。
  - **解析步骤**：
    1. **电路交换计算**：1 Mbps / 100 kbps = **10** 个用户。
    2. **统计多路复用**：分组交换利用了用户活跃概率 p=0.1 的特性，按需分配资源。
    3. **结论**：当支持如 N=35 个用户时，超过 10 个用户同时活跃的概率极小，因此分组交换能支持远多于 10 个的用户。
- **计算题/简答题**: 2Mbps 链路，3 个用户共享；每个用户活跃时以 1Mbps 发送，且只活跃 20% 时间。
  - **解析步骤**：
    1. **电路交换**：每个活跃用户需要 1Mbps，2Mbps 链路最多支持 **2** 个并发用户。
    2. **分组交换**：若同时活跃用户数不超过 2，总输入速率不超过 2Mbps，基本无排队；若 3 个用户同时活跃，总输入速率为 3Mbps，大于链路速率，会产生排队。
    3. **概率口径**：某指定用户活跃概率为 **0.2**；3 个用户同时活跃概率为 $0.2^3=0.008$，也就是队列增长的时间比例约为 **0.8%**。

## 1.3.2 电路交换
### 重要性
★★★★
### 要背的边角料
- **电路 (Circuit) / 端到端连接 (End-to-end Connection)**: 电路交换在通信前建立连接，沿路径交换机维护连接状态，并为该会话预留恒定传输速率。
- **三大阶段**: 建立连接 $\rightarrow$ 数据传输 $\rightarrow$ 释放连接。
- **资源预留**: 独占。包括 **FDM (频分复用)** 和 **TDM (时分复用)**。
- **FDM vs TDM**: FDM 按频带划分，每个连接占用一个频段，频段宽度称为**带宽 (Bandwidth)**；TDM 按时间划分为**帧 (Frame)** 和**时隙 (Time Slot)**，每个连接在每个帧中周期性获得固定时隙。
- **TDM 相比 FDM 的优点**: TDM 更适合数字数据，按时隙分配更容易由数字电路实现；不需要为相邻频带预留保护带，频谱划分开销更少。对于TDM
一条电路的传输速率等于帧速率乘以一个时隙中的比特数量。例如，如果链路每秒传输
8000个帧，每个时隙由8个比特组成，则每条电路的传输速率是64kbps
- **缺点**: 存在 **静默期 (Silent Period)** 导致的资源浪费（由于资源预留，空闲时他人无法使用）。

### 最可能怎么考
- **填空题**: 电路交换通过 **FDM** 或 **TDM** 来实现链路资源的共享。
- **简答题**: 简述电路交换和分组交换在资源分配机制上的本质区别。
  - **答案**：电路交换在通信前必须建立连接并为整个会话**预留/独占**链路资源，即使无数据传输也不释放；而分组交换不预留资源，采用**统计多路复用**按需共享链路带宽，更加高效地适应突发性数据流量。
- **简答题**: 在电路交换中，TDM 相比 FDM 有什么优点？
  - **答案**：TDM 按时间时隙分配链路，更自然地适配数字通信和分组化数据；FDM 需要把频谱划成多个频带并预留保护带，频谱管理和保护带开销更明显。
- **简答题**: 与分组交换相比，电路交换的优点是什么？
  - **答案**：电路交换在连接建立后预留端到端资源，因此可以提供稳定、可预测的传输速率和时延，适合持续、恒定速率的通信；缺点是静默期也占用资源，突发数据场景下利用率较低。

## 1.3.3 网络的网络
### 重要性
★★★★
### 要背的边角料
- **IXP (因特网交换点)**: 多个 ISP 在此对等交换流量。
- **接入 ISP**: 端系统通常通过住宅、企业、大学或移动接入 ISP 接入因特网。
- **区域 ISP / 第一层 ISP**: 接入 ISP 可连接到区域 ISP，区域 ISP 再连接到全球覆盖的第一层 ISP；第一层 ISP 彼此互联。
- **客户-提供商关系 (Customer-Provider Relationship)**: 较低层 ISP 通常向较高层 ISP 付费以获得全球互联；第一层 ISP 位于顶部，通常不向上游付费。
- **第一层 ISP 口径**: 第一层 ISP 位于层级顶部，不向任何上游 ISP 付费；其地位没有正式机构认证，通常凭全球覆盖和互联关系判断。
- **PoP (Point of Presence)**: 提供商网络中供客户 ISP 接入的位置，通常包含一组路由器。
- **PoP 层级位置**: PoP 存在于除最底层接入 ISP 之外的各级提供商网络中。
- **多宿 (Multi-home)**: 一个客户 ISP 或内容提供商连接到两个或多个上游提供商，以提高可靠性或降低成本；即使其中一个提供商故障，也能继续连通因特网。
- **内容提供商网络**: 如 **谷歌专用网络**。
- **网络结构 5**: 在接入 ISP、区域 ISP、第一层 ISP、PoP、多宿、对等和 IXP 之上，再加入内容提供商网络；这是更接近现今因特网的结构。
- **Google 数据**: 拥有 **19 个**主数据中心。
- **对等 (Peering)**: 同级 ISP 通常直接互联并交换彼此客户流量，以减少经过上层 ISP 的费用并改善性能。
- **IXP 盈利方式**: IXP 通常向接入并在此交换流量的 ISP 收取接入、端口或机柜等相关费用。
- **内容提供商自建网络动机**: 降低向 ISP 付费的传输成本，控制端到端服务质量，并把内容更靠近用户以降低时延。

### 最可能怎么考
- **填空题**: 谷歌在全球拥有 **19** 个主数据中心。
- **填空题**: 多个 ISP 在一个共同地点对等交换流量的设施称为 **IXP**。
- **简答题**: 为什么同层级 ISP 倾向于互相对等？IXP 的作用是什么？
  - **答案解析**：同级 ISP 通过对等连接直接交换各自客户之间的流量，可以减少向上级 ISP 付费并降低路径时延。IXP 为多个 ISP 提供集中互联地点，使它们能更方便地建立对等关系并交换流量。
- **应用题**: 分组交换 vs 电路交换：为什么分组交换能支持远多于电路交换的用户？
  - **答案解析**：电路交换必须按每个用户的峰值速率预留资源，因此链路容量固定时可支持用户数受硬上限约束；分组交换使用**统计多路复用**，只在用户实际发送数据时占用链路。若每个用户活跃概率很低，同时活跃用户数超过链路容量的概率也很低，所以能支持更多用户；代价是活跃用户过多时会产生排队、时延甚至丢包。

# 1.4 分组交换网中的时延、丢包和吞吐量
## 1.4.1 分组交换网中的时延
### 重要性
★★★★★
### 要背的边角料
- **节点总时延**: $d_{nodal} = d_{proc} + d_{queue} + d_{trans} + d_{prop}$。
- **传输时延**: **$L/R$** (将分组的所有比特推向链路所需的时间)。
- **传播时延**: **$d/s$** (一个比特从链路一端传播到另一端所需的时间)。
- **传播速率口径**: 典型范围约为 $2\times 10^8$ 到 $3\times 10^8$ m/s；传播时延取决于距离和介质传播速率，不取决于分组长度或链路带宽。
- **处理时延**: 检查分组首部、决定输出链路等处理所需时间，通常较小但不可忽略。
- **排队时延**: 分组在输出缓存中等待发送的时间，随网络拥塞程度变化最大。
- **固定/可变口径**: 处理、传输、传播时延在给定条件下通常较固定；排队时延随流量强度变化，是最不稳定的部分。
- **🚨 致命单位陷阱 (408必考)**: 
  - 数据量 $L$: 1 Byte = 8 bits；1 MB = $2^{20}$ Bytes $\approx 10^6$ Bytes。
  - 链路带宽 $R$: 1 Mbps = $10^6$ bps（永远是逢十进位，绝不是 $2^{20}$）。计算 $L/R$ 时必须先将两者统一转换为 bits 和 bps。

### 最可能怎么考
- **填空题**: 节点总时延由处理时延、排队时延、传输时延和传播时延组成，其中变化最大的是 **排队时延**。
- **简答题**: 请用一句话准确区分传输时延 (Transmission Delay) 和传播时延 (Propagation Delay)。
  - **答案**：传输时延是路由器**将整个分组推出到链路上**所花费的时间（仅与分组长度和链路带宽有关）；而传播时延是一个比特**在物理链路介质中从发送端行进到接收端**所花费的时间（仅与物理距离和介质传播速度有关）。
- **简答题**: 端到端时延中哪些部分较固定，哪些部分变化较大？
  - **答案**：在路径、分组长度和链路速率给定时，处理时延、传输时延和传播时延通常较固定；排队时延取决于路由器缓存前的瞬时流量和拥塞程度，变化最大。
- **应用题/计算题**: 节点总时延综合计算（分组 L=1000 字节，链路带宽 R=10 Mbps，长度 d=2000 km，传播速度 s=2 * 10^8 m/s）。
  - **解析步骤**：
    1. **传输时延 (d_trans)**：L / R = (1000 * 8) / (10 * 10^6) = **0.8 ms**。
    2. **传播时延 (d_prop)**：d / s = (2 * 10^6) / (2 * 10^8) = **10 ms**。
    3. **节点总时延 (d_nodal)**：0.8 + 10 = **10.8 ms**。

## 1.4.2 排队时延和丢包
### 重要性
★★★
### 要背的边角料
- **流量强度**: **$La/R$**，其中 $a$ 是平均分组到达率（packets/s），$L$ 是分组长度（bits），$R$ 是链路传输速率（bps）。
- **趋势**: 若趋近 1，排队时延急剧增加；若 $>1$，趋向无穷大；设计系统时通常应让流量强度不大于 1。
- **排队时延统计量**: 常用平均排队时延、排队时延方差、超过某个时延阈值的概率来描述。
- **丢包原因**: 路由器缓存有限，若分组到达时队列已满，该分组会被丢弃。
- **排队时延计算模板**: 若到达时出链路正在传输一个长度为 $L$ 的分组，且已传出 $x$ bit，队列中已有 $n$ 个完整分组，则新到达分组的排队时延为：

$$
d_{queue}=\frac{L-x}{R}+\frac{nL}{R}
$$

- **流量强度总时延模板**: 若 $I=La/R<1$，且平均排队时延近似为 $\frac{I(L/R)}{1-I}$，则排队加传输总时延为：

$$
d_{queue}+d_{trans}=\frac{L/R}{1-I}
$$

若用 $\mu=R/L$ 表示服务率、$a$ 表示到达率，则：

$$
d=\frac{1}{\mu-a}
$$

### 最可能怎么考
- **判断题**: 当流量强度大于 1 时，分组排队时延是常数。（错误：趋向无穷大）。
- **填空题**: 当路由器缓存已满而新分组到达时，会发生 **丢包**。
- **计算题**: 一个 1500 字节分组到达时，出链路正在发送另一个 1500 字节分组且已发送一半，队列中已有 4 个分组，链路速率 2Mbps，排队时延是多少？
  - **答案解析**：剩余半个分组为 $750\times 8=6000$ bit；队列中 4 个分组为 $4\times1500\times8=48000$ bit。排队时延为 $(6000+48000)/(2\times10^6)=0.027s=27ms$。

## 1.4.3 端到端时延
### 重要性
★★★
### 要背的边角料
- **公式**: $d_{end-end} = N(d_{proc} + d_{trans} + d_{prop})$（假设无拥塞）。
- **逐跳不同参数**: 若各链路速率、距离或处理时延不同，应逐跳相加，而不是硬套一个统一 $N$。
- **多分组流水线**: 多个分组连续经过多条链路时，后续分组可流水线传输；相同速率链路下常见模板为 $(N+P-1)L/R$，再按题意加上传播/处理时延。
- **分组化时延 (Packetization Delay)**: 在 VoIP 等应用中，发送方要先收集足够的数字语音比特填满一个分组，再把分组交给网络；这段收集时间也会影响端到端体验。

### 最可能怎么考
- **计算题**: 计算多跳路径的总时延。
  - **答案解析**：无拥塞、各跳参数相同时，若路径包含 $N$ 条链路，则 $d_{end-end}=N(d_{proc}+d_{trans}+d_{prop})$。其中 $d_{trans}=L/R$，$d_{prop}=d/s$；若各跳参数不同，就逐跳相加：$\sum(d_{proc}+d_{trans}+d_{prop})$。
- **计算题**: 经过 $N$ 条速率均为 $R$ 的链路连续发送 $P$ 个长度为 $L$ 的分组，忽略传播、排队、处理时延，总时间是多少？
  - **答案解析**：第一个分组到达目的端需要 $NL/R$，之后每隔 $L/R$ 到达一个分组，因此 $P$ 个分组全部到达需要 **$(N+P-1)L/R$**。

## 1.4.4 计算机网络中的吞吐量
### 重要性
★★★
### 要背的边角料
- **瞬时吞吐量**: 接收端在某一时刻接收数据的速率。
- **平均吞吐量**: 大小为 $F$ 比特的文件经过 $T$ 秒传输，平均吞吐量为 $F/T$。
- **瓶颈链路**: 端到端路径中传输速率最小的链路，通常决定端到端吞吐量。
- **单连接吞吐量模板**: 若服务器接入链路速率为 $R_s$、客户接入链路速率为 $R_c$，且网络核心足够快，则吞吐量近似为 $\min\{R_s,R_c\}$；现代因特网中接入网常成为限制因素。
- **大文件传输时间估算**: 若吞吐量为 $R_{throughput}$，文件大小为 $F$ bits，则传输时间约为 $F/R_{throughput}$。
- **多连接共享瓶颈**: 多个连接共享同一瓶颈链路时，每个连接实际吞吐量可能低于瓶颈链路速率。
- **多连接共享瓶颈公式**: 若有 $M$ 对客户-服务器共享一条核心链路 $R$，服务器接入链路速率为 $R_s$，客户接入链路速率为 $R_c$，且公平共享核心链路，则每对连接吞吐量为：

$$
\min\left\{R_s,\ R_c,\ \frac{R}{M}\right\}
$$

- **带宽-时延积 (Bandwidth-Delay Product)**: $R\cdot d_{prop}$，表示链路上最多可同时“在途”的比特数，也叫“管道中的比特数”。
- **一个比特的空间宽度**: 若链路长度为 $m$、传播速率为 $s$、带宽为 $R$，则：

$$
\text{bit width}=\frac{m}{R(m/s)}=\frac{s}{R}
$$

### 最可能怎么考
- **填空题**: 端到端吞吐量通常由路径上的 **瓶颈链路** 决定。
- **计算题**: 路径上 3 条链路速率分别为 500kbps、2Mbps、1Mbps，无其他流量，大文件传输吞吐量是多少？
  - **答案解析**：端到端吞吐量由瓶颈链路决定，三者最小为 **500kbps**。若文件大小为 $F$ bits，传输时间约为 $F/500kbps$。
- **简答题**: 带宽-时延积表示什么？
  - **答案**：带宽-时延积 $R\cdot d_{prop}$ 表示发送端连续发送时，链路中同时存在的最大比特数，即链路这根“管道”能容纳多少在途比特。

# 1.5 协议层次及其服务模型
## 1.5.1 分层的体系结构
### 重要性
★★★★★
### 要背的边角料
- **应用层**: 报文 (Message)。
- **运输层**: 报文段 (Segment)。
- **网络层**: 数据报 (Datagram)。
- **链路层**: 帧 (Frame)。
- **物理层**: 比特 (Bit)。
- **典型协议例子**: 应用层有 HTTP、SMTP、DNS (域名系统)；运输层有 TCP (传输控制协议)、UDP (用户数据报协议)；网络层核心是 IP；链路层有 Ethernet (以太网)、WiFi、DOCSIS (电缆数据服务接口规范)。
- **每层服务模型**: 每层通过本层协议向上层提供服务，同时依赖下层提供的服务；层间接口隐藏了下层实现细节。
- **五层主要任务**: 应用层支持网络应用并交换报文；运输层提供进程到进程的数据传送；网络层负责主机到主机的数据报转发与路由；链路层负责相邻节点之间的帧传送；物理层负责在媒介上传输比特。
- **层次可执行的典型任务**: 差错控制、流量控制、分段与重组、多路复用与分用、连接建立等。某些任务可以出现在多个层次，例如链路层和运输层都可能进行差错控制。
- **OSI (开放系统互连) 七层模型对比**: OSI 模型比因特网协议栈多了 **表示层 (Presentation)** (负责加密、压缩、数据转换) 和 **会话层 (Session)** (负责同步、检查点)。在因特网栈中，这两层由**应用层开发者**自己决定是否实现。
- **设备处理的层次**: 主机处理五层；路由器通常处理物理层、链路层、网络层；链路层交换机通常处理物理层和链路层。

### 最可能怎么考
- **填空题 1**: 运输层的协议数据单元 (PDU) 被称为 **报文段 (Segment)**，而网络层的 PDU 被称为 **数据报 (Datagram)**。
- **简答题**: 网络分层的好处是什么？
  - **答案**：分层把复杂系统拆成相对独立的模块，每层只需通过接口使用下层服务并向上层提供服务。这样便于实现、维护、替换某一层协议，也有利于不同网络技术互操作。
- **简答题**: OSI 参考模型包含了哪两个因特网协议栈没有的层次？它们的主要功能是什么？
  - **答案**：多了**表示层**和**会话层**。表示层负责使通信的应用程序解释交换数据的含义（如数据压缩、加密、格式转换）；会话层负责提供数据交换定界和同步功能（如建立检查点以恢复中断的通信）。
- **填空题**: 路由器通常处理到 **网络层**；链路层交换机通常处理到 **链路层**；主机处理完整的五层协议栈。

## 1.5.2 封装
### 重要性
★★★★
### 要背的边角料
- **过程**: 自顶向下每层增加 **首部 (Header)**。有效载荷字段和首部字段
- **封装结果**: 应用层报文加运输层首部形成报文段；报文段加网络层首部形成数据报；数据报加链路层首部形成帧。
- **解封装**: 接收端自底向上逐层去掉对应首部，并把载荷交给上层协议。

### 最可能怎么考
- **简答题**: 简述封装 (Encapsulation) 的过程。
  - **答案**：1. **应用层**产生原始数据报文 M。2. **运输层**添加首部 H_t，形成**报文段 (Segment)**。3. **网络层**添加首部 H_n，形成**数据报 (Datagram)**。4. **链路层**添加首部 H_l，形成**帧 (Frame)**。每一层将上层 PDU 视为载荷附加本层控制信息。
- **填空题**: 在接收端，每层去掉本层首部并把载荷交给上层，这个过程称为 **解封装**。

# 1.6 面对攻击的网络
## 1.6.1 DoS 与 DDoS
### 重要性
★★★★
### 要背的边角料
- **DDoS (分布式拒绝服务)**: 分布式拒绝服务攻击。防御维度：速率限制、ACL (访问控制列表)、异常检测、流量清洗、黑洞路由。
- **DoS (拒绝服务) 三类典型手段**: 漏洞攻击、带宽洪泛、连接洪泛。
- **恶意软件分类**: **病毒 (Virus)** 需用户主动交互（如点开附件）才能感染；**蠕虫 (Worm)** 利用漏洞自动传播，无需用户交互。被控制的主机集合称为**僵尸网络 (Botnet)**。
- **分组嗅探 (Packet Sniffing)**: 被动记录流经链路的分组副本（广播介质极易受害），防御方法是**加密**。
- **IP伪造 (IP Spoofing)**: 将带有虚假源 IP 地址的分组注入因特网。防御方法是**端点鉴别**。
- **中间人/主动攻击**: 攻击者位于通信路径上时，可以窃听、篡改、伪造、重放、丢弃或阻断双方分组。防御思路是加密、完整性校验和端点鉴别。

### 最可能怎么考
- **填空题**: 能够利用目标主机的漏洞自动进入、而不需要任何明显用户交互的恶意软件被称为 **蠕虫 (Worm)**。
- **填空题**: 被攻击者控制、可用于发动 DDoS 的大量受害主机集合称为 **僵尸网络 (Botnet)**。
- **简答题 1**: 简述“IP伪造 (IP Spoofing)”的原理及防范思路。
  - **答案**：原理是攻击者通过修改 IP 数据报的头部，将带有**虚假源 IP 地址**的数据报发送入网络，伪装成受信任主机。防范思路通常是采用**端点鉴别机制**（如使用加密签名或 MAC (消息认证码) 验证数据报的真实来源）。
- **简答题 2**: 设计一个网络层防御 DDoS 攻击的 5 点模板。
  - **答案**：1. **速率限制 (Rate Limiting)**：限制特定流量最高速率。2. **ACLs (访问控制列表)**：丢弃已知攻击特征的分组。3. **异常流量检测**：拦截偏离基准的突发流量。4. **流量清洗 (Traffic Scrubbing)**：重定向可疑流量剔除恶意分组。5. **黑洞路由 (Blackhole Routing)**：牺牲受攻击目标以保护主干网。
- **简答题 3**: 如果攻击者位于 Alice 和 Bob 的通信路径中，能够做哪些恶意事情？
  - **答案**：攻击者可以被动嗅探双方分组，也可以主动篡改报文内容、伪造报文、重放旧报文、丢弃或阻断报文，从而破坏机密性、完整性和可用性。

# 1.7 计算机网络和因特网的历史
## 1.7.1 分组交换的发展：1961～1972
### 重要性
★★
### 要背的边角料
- **先驱**: Leonard Kleinrock (排队论), Paul Baran, Donald Davies。
- **ARPAnet (阿帕网)**: 第一个分组交换网络 (1969)。
- **1972**: Ray Tomlinson 编写第一个电子邮件程序。

### 最可能怎么考
- **填空题**: 1972 年，Ray Tomlinson 编写了第一个 **电子邮件** 程序。

## 1.7.2 专用网络和网络互联：1972～1980
### 重要性
★★
### 要背的边角料
- **核心人物**: Vinton Cerf and Robert Kahn (因特网之父)。
- **协议演进**: TCP/IP 体系结构萌芽；IP 从 TCP 中分离。

### 最可能怎么考
- **填空题**: Vinton Cerf 和 Robert Kahn 被公认为 **因特网之父**。

## 1.7.3 网络的激增：1980～1990
### 重要性
★★
### 要背的边角料
- **1983-01-01**: TCP/IP 正式替代 NCP (网络控制协议) 成为标准。
- **DNS**: 研制用于名字到 IP 映射。

### 最可能怎么考
- **填空题**: **1983** 年 1 月 1 日，TCP/IP 正式取代 NCP 成为标准。

## 1.7.4 因特网爆炸：20 世纪 90 年代
### 重要性
★★

- **WWW (万维网)**: Tim Berners-Lee 发明 HTML (超文本标记语言), HTTP, 服务器和浏览器。
- **浏览器之战**: Mosaic -> Netscape -> Microsoft。

### 最可能怎么考
- **填空题**: 万维网 (WWW) 的发明者是 **Tim Berners-Lee**。
