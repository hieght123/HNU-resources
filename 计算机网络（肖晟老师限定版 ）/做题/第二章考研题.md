## 王道正式保留的题

> [!tip] 王道 PDF 页码速查
> 页码按 `2027计算机网络_高清带书签版_可搜索.pdf` 的 PDF 阅读器页码，不按书本印刷页码。第二章对应王道第 6 章应用层：DNS 在 **PDF p.147-p.150**，电子邮件在 **PDF p.160-p.161**，HTTP/万维网在 **PDF p.169-p.171**。

### DNS + HTTP

- [ ] **PDF p.149，选择 14**：`2020 统考真题`。DNS 查询 + HTTP 访问综合，RTT = 10ms，问从点击 `http://www.abc.com/index.html` 到收到页面的最短/最长时间。
- [ ] **PDF p.149，综合应用 01**：DNS 为什么通常用 UDP；DNS 分组丢失时如何依靠应用层重试/超时处理，什么时候改用 TCP。
- [ ] **可选：PDF p.148-p.149，6.2 选择 12**：DNS 查询最少/最多次数。只用于补递归/迭代边界，时间紧可跳。

### HTTP / 万维网

- [ ] **PDF p.169，选择 11**：非持久 HTTP/1.0 + 浏览器支持并行 TCP，1 个 Web 页引用同网站 7 个小图像，问至少多少 RTT。
- [ ] **PDF p.170，选择 13**：HTTP/1.1 流水线 + DNS 递归/迭代 + 7 个小图像，问从点击超链接到接收全部内容最多多少 RTT。
- [ ] **PDF p.170，选择 17**：`2015 统考真题`。给 HTTP 请求报文，判断 URL、版本、连接方式、Cookie 等叙述。
- [ ] **PDF p.171，综合应用 02**：给 RTT、对象数、对象发送时间，比较非持续连接、持续非流水线、持续流水线的总时间。
- [ ] **可选：PDF p.171，6.5 选择 19**：`2024 统考真题`。非持久 HTTP/1.0，不支持并行 TCP，1 个 Web 页 + 7 个小图像，问 RTT 数。

### 电子邮件

- [ ] **PDF p.161，选择 08**：`2012 统考真题`。用户 1 给用户 2 发邮件，图中 ①②③ 分别用什么协议，重点是 SMTP / SMTP / POP3 或 IMAP/HTTP 的位置。
- [ ] **MIME 简答 1 道**：MIME 与 SMTP 的关系。关键词是：SMTP 限制 7 比特 ASCII，MIME 把非 ASCII 文本/二进制附件编码成可经 SMTP 传输的格式。

### P2P 文件分发

- [ ] **P2P 分发时间计算 1 道**：比较客户-服务器模式和 P2P 模式的最小分发时间下界，能代入 $F$、$N$、$u_s$、$u_i$、$d_i$。
- [ ] **BitTorrent 简答自测**：不用找散题，能说出 tracker、chunk、rarest first、tit-for-tat 的作用即可。

## 不写进王道题单的内容

- **FTP**：第八版第 2 章不是本轮期末重点，王道 FTP 题先跳过。
- **TCP 序号、拥塞窗口、四次挥手**：归到第 3 章。
- **ARP、IP、以太网帧、跨层完整上网流程**：归到第 4 章以后；本次期末只取第 4 章数据平面相关部分。
- **太散选择题**：如果不能训练应用题或简答题模板，不写。
- **综合题保留条件**：只保留 HTTP RTT、DNS + HTTP、P2P 分发时间这种会变成期末应用题的综合题。

## 简答题候选

- [ ] HTTP 为什么是无状态协议？Cookie 如何在无状态 HTTP 上建立用户会话？
- [ ] 非持续 HTTP、持续 HTTP、流水线 HTTP 的 RTT 差异是什么？
- [ ] HTTP/1.1、HTTP/2、HTTP/3 分别如何处理 HOL 阻塞？
- [ ] SMTP、HTTP、IMAP 在邮件发送/取回路径中分别负责哪一段？
- [ ] DNS 为什么采用层次式分布数据库？本地 DNS、根、TLD、权威服务器各做什么？
- [ ] DNS 通常什么时候用 UDP，什么时候用 TCP？
- [ ] P2P 文件分发为什么具有自扩展性？BitTorrent 的 rarest first 和 tit-for-tat 解决什么问题？
- [ ] DASH 客户端怎样根据告示文件和可用带宽选择视频块？CDN 为什么能降低时延？

## 应用题候选

- [ ] **DNS + HTTP 总时延**：画出 DNS 查询链，再加 TCP 建连、HTTP 请求/响应的 RTT。
- [ ] **HTTP 多对象下载时间**：区分非持续、持续非流水线、持续流水线、并行 TCP。
- [ ] **HTTP 请求报文判断**：方法、URL、版本、Host、Connection、Cookie、CRLF。
- [ ] **邮件协议路径**：Web 邮箱/客户端发信、服务器中继、收信分别用 HTTP、SMTP、IMAP/POP3。
- [ ] **P2P 分发时间**：会写 $D_{cs}$ 和 $D_{p2p}$ 的下界公式，并能代入 $F$、$N$、$u_s$、$u_i$、$d_i$。
